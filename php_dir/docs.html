1. Abstract

Resource Web Planner este o aplicație web care permite utilizatorilor să își administreze mai multe inventare, fiecare inventar conținând mai multe resurse și fonduri.

Acest raport tehnic va descrie obiectivele proiectului, tehnologiile folosite în cadrul acestui proiect, exemple de utilizare ale aplicației structura internă a aplicației și îmbunătățiri posibile.

2. Introducere

2.1. Cerința

Să se dezvolte o aplicație Web care permite gestionarea pe categorii a articolelor esentiale și consumabilelor (e.g., becuri, lemne pentru foc, condimente, toner, cosmetice, pahare de unică folosință, pioneze, medicamente de uz general,…), plus a pieselor de schimb pentru diverse dispozitive – de exemplu, aparate electr(on)ice, automobile, ascensoare etc. – dintr-o gospodărie, organizație sau întreprindere. Instrumentul va fi capabil să notifice, periodic sau la date fixe (eventual, prestabilite), utilizatorul sau un grup de utilizatori în ceea ce privește epuizarea iminentă a unor materiale sau verificarea anumitor echipamente și angrenaje. Aceste notificări vor fi semnalate atât direct în cadrul interfeței aplicației, cât și prin poșta electronică. Se vor oferi posibilități de importare/exportare a datelor în formatele CSV, JSON si XML, iar diversele statistici generate vor putea fi consultate sub forma de documente HTML și PDF. Link

Cele mai importante aspecte pentru fiecare proiect în parte sunt:

  • Rezolvarea integrală a problemei – conform cerinţelor specifice fiecărui proiect propus.

  • Proiectele vor putea fi implementate pe partea de server folosind serverul de aplicaţii şi limbajul de programare PHP.

  • Nu se permite utilizarea de framework-uri la nivel de client (front-end) şi/sau server (back-end) Web.

  • Architectura aplicaţiei Web realizate va fi obligatoriu bazată pe servicii Web. Invocarea unui serviciu/API Web va fi efectuată în manieră asincronă via suita de tehnologii Ajax.

  • Pentru partea de client, interfaţa aplicaţiei Web va fi marcată obligatoriu în HTML – codul trebuind să fie valid conform specificaţiilor Consorţiului Web. Se vor utiliza foi de stiluri CSS valide – pentru verificare, se poate recurge la instrumente dedicate precum Stylelint.

  • Pentru stocarea şi managementul datelor, se vor putea utiliza servere de baze de date relaţionale, interogate via SQL – minimal, a se considera SQLite. Complementar, se poate recurge la servere de baze de date aliniate paradigmei NoSQL.

  • Se vor folosi pe cât posibil machete (template-uri) de prezentare şi metode de configurare şi administrare a aplicaţiei.

  • Alte cerinţe obligatorii:

    • Adoptarea principiilor designului Web responsiv.

    • Recurgerea la tehnici de prevenire a atacurilor (minimal, SQL injection şi Cross Site Scripting).

    • Import/export de date folosind formate deschise – minim, CSV (Comma Separated Values) şi JSON (JavaScript Object Notation).

    • Existenţa unui modul propriu de administrare a aplicaţiei Web.

2.2. Obiective

Noi am interpretat această cerință în felul următor:

Se dorește o aplicație web care să permită utilizatorilor să își administreze mai multe inventare, fiecare inventar conținând mai multe resurse și fonduri. Pe baza acestor date, se pot exporta diverse statistici în HTML și PDF, iar inventarele să poată fi importate/exportate în formatele CSV, XML și JSON.

Resursele ar trebui să poată fi organizate după tipul lor, ceea ce ne-a dus cu gândul la a avea tag-uri create de către utilizator.

Aplicația să poată trimite notificări utilizatorilor (și în interfață și prin mail) în cazul unei epuizări iminente a unei anumite resurse. Noi am implementat această cerință în felul următor: fiecare resursă din cadrul fiecărui inventar are un threshold critic (by default 0). Când cantitatea acestei resurse este mai mică sau egală cu threshold-ul, se generează o notificare.

Pe lângă aceste obiective identificate, avem niște funcționalități suplimentare, enumerate aici.

3. Tehnologii folosite

  • Web: PHP, HTML, CSS, JS
  • JWT
  • Deployment: Docker
  • Baze de date: PostgreSQL, SQL

4. Exemple de utilizare și funcționalități

Funcționalități:

  • Login, Register, Change Password, Forgot Password.
  • Crearea și ștergerea inventarelor.
  • Permisiuni granulare asupra inventarelor (Read, Edit, Update, Delete).
  • Import și export a inventarelor în formatele CSV, XML și JSON.
  • Crearea și ștergerea de resurse în cadrul unui inventar.
  • Crearea tranzacțiilor în cadrul unui inventar (schimbarea cantității unei resurse).
  • Trimis mail + notificare în aplicație când cantitatea unei resurse scade sub un anumit prag.
  • Trimis mail cand se modifică anumite permisiuni ale unui user, respectiv când se șterge un inventar.
  • Organizarea resurselor pe tag-uri create de utilizator.
  • Un istoric al tuturor tranzacțiilor.
  • Crearea unui raport (HTML, PDF) al tuturor tranzacțiilor pentru o anumită resursă într-un anumit interval de timp.
  • O listă cu toate valutele raportate la USD.
  • Un panel pentru admini.
  • Prevenție XSS (folosind htmlspecialchars()) și SQL Injection (folosind proceduri stocate).
  • Front-end responsiv folosind media queries și valori relative (eg. 90%) pentru dimensiunea elementelor.

5. Structura internă a aplicației

Aplicația constă din 3 imagini Docker:

  • O imagine creată prin maparea subfolder-ului /php_dir în Docker.
  • O imagine pentru baza de date (PostgreSQL).
  • O imagine pentru PGAdmin.

Containerele rezultate vor comunica prin cadrul rețelei rwp_net din cadrul Docker.

Arhitectura aplicației este bazată pe modelul MVC. Subfolderul /php_dir/app conține la rândul său 4 subfoldere:

  • /php_dir/app/models: folderul care conține toate modelele.
  • /php_dir/app/views: folderul care conține toate view-urile.
  • /php_dir/app/controllers: folderul care conține toate controllerele.
  • /php_dir/app/services: folderul care conține toate serviciile.
Pe lângă acestea, mai există următoarele foldere:
  • /php_dir/admin_scripts: conține scripturile de populare a bazei de date.
  • /php_dir/style: conține toate fișierele CSS folosite.
  • /php_dir/assets: folderul care conține toate asset-urile (favicon + un clopot inactiv nefolosit).
  • /php_dir/js: folderul care conține toate scripturile Javascript folosite.
Toate celelalte fișiere PHP din php_dir/ sunt menite să fie accesate de către utilizatorii aplicației.
